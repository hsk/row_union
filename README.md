# Row-Union 列多相Unionと積型データコンストラクタによる型システム・インタプリタ

このプロジェクトは、**列多相 Union (Row polymorphic Union)** の toy言語のプロジェクトです。
**列多相 Union** は **列多相バリアント(Row polymorphic variants)** を変形しタグがない値も扱えるようにすることでより簡潔な表現を可能にしたものです。
さらに **積型（Product Types）としてデータコンストラクタ** を組み合わせ **再帰型(Recursive Types)** にも対応することで、柔軟かつ型安全なオープン・バリアント（拡張可能な直和型）を実現しました。

このプロジェクトには、２つの実装が含まれています。１つはコンセプトの数理的な検証（プロトタイプ）として記述された **Prolog 実装**です。
もう一つが実用的なパーサ・型チェッカー・評価器（インタプリタ）を備えた **OCaml 実装**です。

---

## 🚀 主な特徴と理論的背景

### 1. 列多相Union (Row polymorphic Union)

本システムにおける型は、内部的には「列（Row）」のリスト（Union）として表現されます（OCamlコード内では `type t = typ list`）。

- **包含関係によるサブタイピング**:
  列としての表現により、Union型の包含関係に基づく直感的なサブタイピングが自然に定まります。
  例えば、`[Int; Bool]` は `[Int; Bool; String]` の部分型です(`[Int; Bool] <: [Int; Bool; String]`)。
  しかし、具象構文では `Int | Bool <: Int | Bool | String` のように扱います。
- **関数のサブタイピング**:
  関数型 `A -> B` では、引数型が反変 (contravariant)、戻り値型が共変 (covariant) となる標準的なサブタイピング関係を安全に解決します。

### 2. 積型としてのデータコンストラクタ (Product-typed Data Constructors)

`Add(l, r)` のようなデータコンストラクタ（`Const`）は、任意の引数（積型、リスト表現）を保持するコンストラクタとして構成されます。

- **共変サブタイピング**:
  コンストラクタは引数の積型に対して共変（covariant）に振る舞います。
  例えば、`Add(A,A) <: Add(B,B)` は `A <: B` であるとき、またそのときに限り成り立ちます。
- **再帰型（Recursive Types）の自動解決**:
  列多相Unionと積型コンストラクタの組み合わせにより、型定義を事前に明示することなく、構文木やリストなどの再帰的なデータ構造（自己参照する列多相Union）を型推論中に不動点（サイクル検出付きの再帰的な単一化）として自動的に解決します。

### 3. 高度なパターンマッチング

- ネストしたコンストラクタパターン (`Add(Add(l1, r1), r)`) や、実行時の列多相Unionに対する型注釈パターン（`i:Int`）に対する静的型チェックおよびパターンマッチが可能です。

## 📝 言語仕様と構文例

`example/test.row` にあるプログラムは、以下のような構文で記述されます。

```ocaml
(* 1. 型付き・コンストラクタを混在したパターンマッチ *)
let rec ev x =
  match x with
  | Add(l, r) -> ev l + ev r
  | i:Int -> i

(* 2. ネストした積型コンストラクタのパターンマッチ *)
let rec ev2 x =
  match x with
  | Add(Add(l1, r1), r) -> ev2 l1 + ev2 r1 + ev2 r
  | i:Int -> i

(* 3. 変数定義と、動的に推論される再帰的なコンストラクタ値の適用 *)
let v = 1
let v = Add(Mul(2,1),3)
let v = ev (Add(Add(1,2),Add(3,4)))
```

と実行すると以下のように型推論と評価の結果が得られます:

```ocaml
val ev : (Add('c, 'c) | Int) as 'c -> Int = <fun>
val ev2 : (Add(Add('c, 'c), 'c) | Int) as 'c -> Int = <fun>
val v : Int = 1
val v : Add(Mul(Int, Int), Int) = Add(Mul(2, 1), 3)
val v : Int = 10
```

### 型推論とサブタイピングの例

上記の `ev` 関数を型検査すると、引数 `x` は以下のような再帰型として推論されます。

- 引数 `x`: `Int | Add('a, 'a) as 'a` (ここで `'a` は自分自身を表す型変数)
- 戻り値: `Int`

これにより、明示的に型定義（代数的データ型）を宣言することなく、`Add(Add(1, 2), 3)` のようなネストした木構造のデータを安全に扱い、列多相Unionによる柔軟な型検査を行うことができます。

## 🔍 実装の技術的詳細

### トランザクション機能付き単一化（OCaml）

`typing.ml` には `with_transaction` 補助関数が実装されています。サブタイプ関係（`subtype`）を走査する際、一時的な型のバインド（`set_link`）を行い、判定が失敗した場合には自動でロールバック（巻き戻し）を行います。これにより、非決定的な部分型探索を副作用のある単一化で効率よく実現しています。これは並行制御するSTMではなく、型推論の探索における『試行と放棄』を安全に管理するためのロールバック・トレイトです。

### 不動点とサイクル検出による再帰型解決

再帰的な部分型判定が無限ループに陥るのを防ぐため、`seen` リスト（履歴）を用いて共帰的（coinductive）に同一性を検査するサイクル検出アルゴリズムが組み込まれています。
